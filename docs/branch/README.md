# Branch Notes: dev

## Purpose
The scratchpad. For raw C++ header templates, boilerplate, and experimental implementations not yet ready for a named feature branch.

## Scope
Experimental code, WPILib API exploration, header stubs, placeholder subsystem files, and anything that helps prototype ideas without polluting a named branch.

## Key Responsibilities
- Providing reusable header templates for new subsystems
- Experimenting with WPILib APIs before committing to a design
- Storing one-off utility scripts and test harnesses

## Workflow
Code that proves useful should be promoted to a named feature branch rather than living here long-term. Dev is not merged into `main`.
